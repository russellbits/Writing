# Dice Box — App Specification

## Overview

A full-screen immersive 3D experience: a felt-lined box rendered in Three.js, containing a single orange dodecahedral die with white numerals 1–20. The user tilts and shakes their phone to roll the die via the device Gyroscope/Accelerometer API. Built as a Svelte web app, packaged for iOS via Capacitor.

---

## Tech Stack

| Layer | Choice | Notes |
|---|---|---|
| UI Framework | **Svelte 5** | Minimal overhead, ideal for a single-component app |
| 3D Rendering | **Three.js** | Scene, camera, lighting, geometry, felt texture |
| Physics | **Rapier3D** (WASM) | Best perf/accuracy for web; supports convex hull colliders |
| Native Bridge | **Capacitor 6** | Motion API plugin for gyro/accel |
| Build Tool | **Vite** | Fast HMR, Capacitor-compatible |
| Target | **iOS** (web fallback) | `@capacitor/motion` → `DeviceMotion` API fallback for browser |

> **Why Rapier?** Rapier is a Rust-compiled WASM physics engine. It handles convex polyhedra accurately and performs well on mobile. Alternatives: Cannon-ES (simpler, less accurate for polyhedra), Ammo.js (heavy).

## Viewport & Sizing

- The Three.js `<canvas>` is **absolutely positioned, 100dvw × 100dvh**, filling the safe area of the device screen edge-to-edge (no margins, no UI chrome).
- Camera is **perspective** (preferred, ~60° FOV) pointing straight down into the box. The box depth gives a convincing "looking into a box" feel — approximately 1:1.5 aspect (width:depth).
- **Die diameter** = `min(viewportWidth, viewportHeight) * 0.25`. Calculated at mount time and on `resize`. The Three.js dodecahedron radius is set to half this value.
- The box interior width/height matches the viewport exactly (walls are at the edges of the canvas). Box depth is ~1.5× the die diameter.

---

## The Box

- **Floor and 4 walls** are flat `PlaneGeometry` / `BoxGeometry` meshes.
- Material: a **felt-like MeshStandardMaterial** with:
  - Deep green (`#1a4a2e`) base color
  - `roughness: 1.0`, `metalness: 0.0`
  - Optional: a subtle procedural noise bump map for fabric texture
- The box is **static** in the physics world — all 5 surfaces are static Rapier colliders (floor + 4 walls). Walls are invisible from the camera angle but present as physics boundaries.
- The **top is open** — the die cannot escape; if it somehow exits, respawn it in center.

## The Die

### Geometry

- `DodecahedronGeometry(radius, 0)` — 12 pentagonal faces
- Orange base color: `#FF6B1A` (vivid, slightly warm)
- `roughness: 0.4`, `metalness: 0.1` — slight sheen, not plastic
- Each of the 12 faces gets a **white numeral** (see note below)

> **Note on "dodecahedral d20":** A standard d20 is an *icosahedron* (20 faces). A *dodecahedron* has 12 faces. The spec calls for a dodecahedron numbered 1–20 — this means some numbers repeat or it's stylized. **Clarification needed from owner before implementation:** should this be a true d12 (1–12) or a dodecahedron *styled* as a d20 (1–20 spread across 12 faces)? For now, spec assumes **1–12 numbered faces** with the "d20" label being aspirational flavor.

### Face Numerals

- Each pentagonal face receives a `CanvasTexture` with the numeral centered, white, bold serif font (e.g. Palatino or a custom loaded font)
- Texture resolution: 256×256 per face
- UV mapping: faces are individually UV-unwrapped (Three.js `DodecahedronGeometry` groups = 12 face groups)

### Physics Body

- Rapier `RigidBodyDesc.dynamic()`
- Collider: `ColliderDesc.convexHull(vertices)` — extract vertices from Three.js geometry
- Restitution: `0.4` (moderate bounce), Friction: `0.6` (felt-like)
- Initial position: center of box, slightly above floor
- Initial velocity: zero; gravity pulls it down on load

## Motion / Gyroscope Input

- On iOS via Capacitor: `@capacitor/motion` plugin — `Motion.addListener('accel', callback)`
- Fallback (web/Android): `window.addEventListener('devicemotion', callback)`
- The **accelerometer vector** (x, y, z in m/s²) maps directly to Rapier's gravity vector:
  `world.gravity = { x: accel.x * scale, y: accel.y * scale, z: 0 }`
- `scale` factor: tune so that full tilt (≈9.8 m/s²) produces vigorous but not frantic die movement. Starting value: `2.0`
- **Permission gate:** iOS 13+ requires `DeviceMotionEvent.requestPermission()`. A tap-to-start overlay handles this — single tap on the canvas requests permission and removes the overlay.

---

## Game Loop

```
requestAnimationFrame loop:
  1. Step Rapier physics world (fixed timestep 1/60s, with interpolation)
  2. Sync Three.js die mesh position + quaternion ← Rapier rigid body
  3. renderer.render(scene, camera)
```

### iOS-Specific

- Add `NSMotionUsageDescription` to `Info.plist`: `"This app uses motion sensors to roll the die."`
- Disable screen rotation lock handling (app should work in portrait only initially, or support both orientations by recalculating box dimensions on orientation change)
- `WKWebView` configuration: ensure `allowsInlineMediaPlayback` and `mediaTypesRequiringUserActionForPlayback` are set for any audio if added later

## Interaction Design

| Trigger | Behavior |
|---|---|
| First tap (iOS) | Request DeviceMotion permission, show die, start physics |
| Tilt phone | Gravity shifts, die rolls toward low side |
| Shake (rapid accel spike) | Apply impulse to die: `body.applyImpulse({ x, y, z })` |
| Double-tap | Re-center die (teleport to center, zero velocity) |
| Resize / orientation change | Recalculate viewport, resize renderer, resize box colliders, rescale die |

## Open Questions / Decisions for Owner

1. **d12 vs d20:** 12 faces numbered 1–12, or 1–20 on 12 faces (some numbers skipped/repeated)?
2. **Orientation support:** Portrait only, or landscape too (box becomes wider)?
3. **Sound:** Felt-muffled rolling sound on die contact with walls/floor?
4. **Result display:** Show the top face value in a UI overlay after the die settles?
5. **Native iOS vs Capacitor:** Capacitor is lower friction. A native SwiftUI + SceneKit or RealityKit implementation would be smoother but far more effort.
